# 🔄 4. Data Sync Protocol (LAN)

## 4.1 Connection Architecture
The Mobile App (APK) and the Desktop POS (Server) communicate over a **Local Area Network (LAN)** using HTTP. This local-first approach ensures speed and reliability without needing constant internet access.

### 🌐 IP Addressing
- **Desktop (Server):** Runs on a dynamic local IP (e.g., `192.168.100.7`). This IP can change if the router reassigns it.
- **Port:** The server listens on Port `3000`.
- **Protocol:** HTTP (Not HTTPS) is used for local LAN communication.
- **Example URL:** `http://192.168.100.7:3000`

### 🛡 Android Cleartext Traffic
Since the connection is HTTP (unencrypted local traffic), Android requires explicit permission to allow "Cleartext Traffic".
**Configuration:**
- File: `res/xml/network_security_config.xml`
- Content:
  ```xml
  <network-security-config>
      <base-config cleartextTrafficPermitted="true" />
  </network-security-config>
  ```
- Manifest: `<application android:networkSecurityConfig="@xml/network_security_config" ...>`

## 4.2 Required Permissions
To function correctly, the APK requires the following Android permissions:

1.  **INTERNET:** Required to communicate with the Desktop Server via the LAN.
    - `<uses-permission android:name="android.permission.INTERNET" />`
2.  **CAMERA:** Required to scan the "Connection QR Code" generated by the Desktop App.
    - `<uses-permission android:name="android.permission.CAMERA" />`

## 4.3 Sync Lifecycle
1. **Connect:** User scans QR code containing `apiUrl` (e.g., `http://192.168.1.15:3000`) and `apiKey`.
2. **Pull:** App fetches initial state from `GET /api/sync`.
3. **Queue:** Offline actions (transactions, expenses) are stored in `syncQueue`.
4. **Push:** App attempts to send queued actions to `POST /api/sync` every 10-60 seconds.

# 🖥 5. Desktop Server Requirements
Desktop POS must:
1. Expose `/api/sync` (batch ops)
2. Handle ops: transaction, expense, product, user, customer
3. Serve images from:
```
http://<IP>:3000/assets/<file>
```
4. Generate QR containing `{ apiUrl, apiKey }`

### 5.1 API Headers & Authentication
The Desktop Server (`electron.cjs`) protects endpoints using an API Key.
Every request from the Mobile App (except `/api/status`) **must** include one of the following headers:

- **X-API-KEY**: `<The API Key>`
- **Authorization**: `Bearer <The API Key>`

The API Key is generated by the Desktop App and stored in `server-config.json`.
